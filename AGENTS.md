# LumenWall 发布规则

以下规则适用于本项目的所有功能开发、打包与发布工作：

1. 每次生成新的安装包后，`dist/` 中只能保留该最新版本的 PKG；应移除旧版本安装包和临时 staging 构建目录，避免误安装或误发布过期二进制。
2. 每完成一个功能或修复，必须进行回归检查，确认新改动不会影响既有的搜索、下载、设为壁纸、收藏、缓存、定时更换、菜单栏、语言与外观等功能。
3. 仅在构建与测试均通过后，才可提交并推送到 GitHub；随后创建对应版本的 GitHub Release，并上传已验证的最新 PKG。Release 说明应注明核心变更和验证结果。

## 最低验证清单

- 使用独立、干净的构建目录重新编译，禁止复用来源不明的旧 `.build` 产物。
- 运行 `swift test`，所有测试必须通过。
- 启动最终将被打包的 `.app`，检查本次改动的界面/行为可用。
- 校验 PKG 中应用的版本号、资源文件与签名，再上传 Release。
